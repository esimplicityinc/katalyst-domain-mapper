# US-076: Manage Capabilities via UI

**As an** Engineering Team Lead,
**I want to** manage capabilities through a web UI,
**so that** I can build and maintain the capability taxonomy that structures our domain landscape without writing markdown files by hand.

## Context

Prior to this feature, capabilities were static markdown files maintained by hand. The CapabilityTreeView component (813 lines) provides a full CRUD interface for the 3-level capability hierarchy (root → subsystem → leaf), backed by new API routes and a SQLite migration (0009). Changes persist immediately to the governance database and flow into the Business Landscape visualization.

## Road Items

- [ROAD-046](../roads/ROAD-046) — Governance Artifact Management UI

## Acceptance Criteria

1. The Architecture page Capabilities tab renders the full 3-level capability hierarchy as an interactive tree
2. Users can create new capabilities with title, description, category, status, taxonomyNode, parentCapability, and tag fields
3. Users can edit existing capabilities inline without leaving the page
4. Users can delete capabilities with a confirmation dialog
5. Capability nodes show status badges (planned/stable/deprecated) and taxonomy node associations
6. Changes to capabilities are immediately persisted to the API and visible in the landscape
