# US-095: Track Team Adoption of Practice Areas

**As an** Engineering Team Lead,
**I want to** track my team's adoption level for each practice area,
**so that** I can measure maturity progress and identify areas for investment.

## Context

Team adoptions record a team's maturity level for a given practice area. Each record captures the team name, practice area, maturity level, supporting evidence, and the assessment date. The by-team query endpoint enables viewing all practice area adoption levels for a single team, providing a maturity dashboard view.

## Road Items

- ROAD-048 (Practice Area, Competency & Adoption CRUD)

## Acceptance Criteria

1. GET `/api/v1/taxonomy/team-adoptions` returns all team adoption records across practice areas
2. GET `/api/v1/taxonomy/team-adoptions/by-team/:teamName` returns all adoption records for a specific team
3. POST `/api/v1/taxonomy/team-adoptions` creates a new team adoption record with maturity level, evidence, and assessment date
4. PUT `/api/v1/taxonomy/team-adoptions/:id` updates an existing team adoption record to reflect maturity progression
5. DELETE `/api/v1/taxonomy/team-adoptions/:id` removes a team adoption record from the system
