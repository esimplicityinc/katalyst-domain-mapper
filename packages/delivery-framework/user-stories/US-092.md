# US-092: Trigger Governance Validation From CI Pipeline

**As a** CI/CD Pipeline,
**I want to** execute governance validation as a pipeline step and receive structured exit codes,
**so that** I can gate merges on governance health and report results to the PR.

## Context

UT-008 (CI/CD Pipeline) defines the automated infrastructure that runs governance validation on commits. The governance-linter.js and validate-docs.js scripts in the delivery-framework partially implement validation, but produce console-only output that isn't easily consumed by CI systems. This story defines the contract for machine-readable validation output with structured exit codes that CI pipelines can parse and act on — blocking merges on integrity violations while allowing warnings for draft artifacts.

## Road Items

- (partial — scripts exist in `packages/delivery-framework/scripts/`, but need structured output)

## Acceptance Criteria

1. `bun run validate:governance` exits with code 0 on success and non-zero on failure
2. The validation output is machine-readable JSON (not just console text)
3. Validation checks include: frontmatter completeness, cross-reference integrity (every `user_type: UT-XXX` references an existing user type), and governance gate status
4. Missing or broken cross-references are reported as structured errors with file paths and line numbers
5. The exit code distinguishes between hard failures (integrity violations) and warnings (draft status, missing optional fields)
