# US-097: View Organization Map

**As a** Domain Architect,
**I want to** view an interactive force-directed graph of teams, persons, systems, and user types,
**so that** I can understand organizational structure, identify Conway's Law alignment issues, and spot cognitive load risks from multi-team membership.

## Context

The Organization Map is a d3-force SVG visualization that surfaces the structural spine of the engineering organization. It reuses the existing `useSvgPanZoom` hook and d3-force layout patterns established in the Context Map Diagram (CAP-023). Nodes are typed (teams as rounded rectangles, persons as circles, systems as hexagons, user types as diamonds) and edges represent three relationship types: member-of (person → team), owns (team → system), and serves (system → user type, derived from user stories linking user types to capabilities that map to bounded contexts). Multi-team membership is immediately visible as a person node with multiple edges.

## Road Items

- ROAD-049 (Organization Intelligence Views)

## Acceptance Criteria

1. The Organization Map page renders a force-directed graph with teams, persons, systems/bounded contexts, and user types as distinct node shapes
2. Team nodes are colored by Team Topologies type (stream-aligned=blue, platform=purple, enabling=green, complicated-subsystem=orange)
3. Person nodes display initials and connect to their team(s) via member-of edges
4. System/bounded context nodes connect to their owning team via owns edges
5. User type nodes connect to systems via serves edges (derived through capability linkage from user stories)
6. Clicking any node highlights its immediate neighborhood and dims unrelated nodes to 0.25 opacity
7. The graph supports pan and zoom via mouse drag and scroll wheel
8. A filter toolbar allows toggling node types on/off and filtering by team type
9. The graph loads within 3 seconds for up to 50 nodes
