# US-102: View Contribution-to-Outcome Traceability

**As an** Engineering Team Lead,
**I want to** trace contributions from people through road items to capabilities to user outcomes,
**so that** I can measure the impact of engineering investments and demonstrate ROI to stakeholders.

## Context

The Outcome Traceability view answers the question "which contributions led to what outcomes?" It reverses the direction of the User Type Journey — instead of starting from users and tracing to teams, it starts from people and their contributions and traces forward to the user value delivered. This makes visible the often-invisible governance work (ADR writing, change documentation, road item progression) by connecting it to concrete user outcomes. The status indicators at each node in the chain reveal where work is blocked and which chains are fully delivered.

## Road Items

- ROAD-049 (Organization Intelligence Views)

## Acceptance Criteria

1. The Outcome Traceability page displays a tree/cascade visualization tracing from persons through contributions to road items to capabilities to user stories to user types
2. Each node in the chain shows its current status (contribution status, road item phase, capability status, story status)
3. Completed chains (where a contribution's road item has delivered a capability that fulfills a user story) are highlighted in green
4. In-progress chains show amber indicators for the blocking node
5. Clicking any node highlights its full upstream and downstream chain
6. A filter panel allows filtering by person, road item, capability, or time range
7. Gap indicators show road items without contributions, capabilities without road items, and stories without capabilities
8. The traceability view loads within 3 seconds
