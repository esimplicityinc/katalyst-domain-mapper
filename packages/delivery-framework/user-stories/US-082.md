# US-082: Override Flags Via Environment Variables in CI/CD

**As a** CI/CD Pipeline,
**I want to** override feature flag values via `FF_*` environment variables,
**so that** pipeline configuration can control feature behavior without modifying `flags.json`.

## Context

CI/CD pipelines need to control feature flags at the environment level — enabling strict linting in staging, disabling the scanner in test environments, or forcing a specific AI runtime for integration tests. The `FF_*` environment variable convention provides a standard, discoverable mechanism for pipeline configuration that takes precedence over `flags.json` defaults. This is the first user story for UT-008 (CI/CD Pipeline), which was previously defined but had zero stories.

## Road Items

- (retroactive — built as part of the feature flags infrastructure)

## Acceptance Criteria

1. Environment variables prefixed with `FF_` override corresponding flags (e.g., `FF_SCANNER_ENABLED=false` overrides `scanner-enabled`)
2. The override uses kebab-to-SCREAMING_SNAKE case conversion (`landscape-layout-v2` → `FF_LANDSCAPE_LAYOUT_V2`)
3. Overrides take precedence over `flags.json` defaults
4. The `/api/v1/flags` endpoint reflects the effective values including overrides
5. Invalid override values (e.g., `FF_MAX_CONCURRENT_SCANS=abc` for a number flag) log a warning and fall back to the default
