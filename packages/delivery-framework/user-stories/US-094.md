# US-094: Manage Competencies via CRUD API

**As a** Platform Engineer,
**I want to** manage competencies through the CRUD API,
**so that** I can define the assessable skills within each practice area and their prerequisite dependencies.

## Context

Competencies represent the assessable skills within a practice area. Each competency belongs to a practice area, has a proficiency level (basic, intermediate, advanced), and may declare prerequisites forming a directed acyclic graph (DAG). The DAG structure ensures learning paths are well-ordered and free of circular dependencies.

## Road Items

- ROAD-048 (Practice Area, Competency & Adoption CRUD)

## Acceptance Criteria

1. GET `/api/v1/taxonomy/competencies` returns a list of all competencies with their practice area associations
2. GET `/api/v1/taxonomy/competencies/:name` returns a single competency by name with full detail including prerequisites
3. POST `/api/v1/taxonomy/competencies` creates a new competency with practice area, proficiency level (basic, intermediate, advanced), and optional prerequisites
4. PUT `/api/v1/taxonomy/competencies/:name` updates an existing competency and persists changes
5. DELETE `/api/v1/taxonomy/competencies/:name` removes a competency from the system
6. Creating or updating a competency with circular prerequisite dependencies returns a validation error (DAG enforcement)
