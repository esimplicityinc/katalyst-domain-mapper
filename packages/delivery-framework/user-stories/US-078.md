# US-078: Use AI Chat for Taxonomy and User Type Discovery

**As an** Engineering Team Lead,
**I want to** discover capabilities, user types, and user stories through guided AI conversation,
**so that** the discovery process surfaces artifacts I might miss when creating them manually — and persists them to the system automatically.

## Context

The `taxonomy-architect` and `user-type-storyteller` AI agents (see ADR-019) are embedded within the Architecture and User Types pages respectively. Unlike standalone chat tools, these agents have access to the current taxonomy state via pre-loaded context and can immediately persist discovered artifacts via API calls — the live UI updates as the conversation progresses.

## Road Items

- [ROAD-046](../roads/ROAD-046) — Governance Artifact Management UI

## Acceptance Criteria

1. The Architecture page Chat tab opens a streaming AI chat with the taxonomy-architect agent
2. The User Types page Chat tab opens a streaming AI chat with the user-type-storyteller agent
3. The taxonomy-architect agent reads the current capability tree and taxonomy snapshot before responding
4. The user-type-storyteller agent follows a 6-phase discovery protocol (discovery → goals/pain points → story writing → Gherkin → capability mapping → coverage review)
5. Both agents auto-persist discovered artifacts to the API via curl during the conversation
6. Newly created artifacts appear in the management UI without requiring a page reload
