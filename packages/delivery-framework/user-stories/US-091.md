# US-091: View Landscape With Zoned Trunk Layout

**As a** Domain Architect,
**I want to** view the business landscape using a zoned trunk layout that separates user-facing and system-supporting contexts into tiers,
**so that** I can quickly distinguish between customer-facing services and internal platform infrastructure.

## Context

The `ZonedLayoutEngine` is the fourth layout engine for the Business Landscape canvas (alongside ELK, Dagre, and D3-Force). It introduces a tiered layout strategy that separates bounded contexts into "User-Facing" (contexts with direct user type interactions) and "System-Supporting" (contexts that serve other contexts). A visual trunk line separates the tiers, and event flow arrows route along the trunk to minimize visual crossing. The engine is gated behind the `landscape-layout-v2` feature flag (CAP-028) for incremental rollout.

## Road Items

- ROAD-044 (Business Landscape Visualization)

## Acceptance Criteria

1. When the `landscape-layout-v2` feature flag is enabled, the "Zoned" layout option appears in the layout engine selector
2. The zoned layout groups bounded contexts into two tiers: "User-Facing" (top) and "System-Supporting" (bottom), separated by a visual trunk line
3. Domain event flow arrows route along the trunk between tiers rather than crossing freely
4. The layout engine respects the same pan/zoom, filtering, and selection interactions as other layouts
5. Switching between layout engines preserves the current filter and selection state
