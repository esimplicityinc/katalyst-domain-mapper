# US-101: Trace User Type Journey Through Teams

**As a** Domain Architect,
**I want to** trace how user types flow through stories, capabilities, systems, and teams,
**so that** I can verify that team boundaries align with user value streams and identify organizational coupling.

## Context

The User Type Journey view is a simplified, user-centric perspective on the data that the Business Landscape canvas renders with full domain detail. Where the landscape shows everything at once, the journey view focuses on the question "who are we building for and which teams deliver it?" The multi-column layout makes the chain of dependencies explicit and surfaces broken chains (capabilities that no system implements, stories that no capability supports) as visual gaps. It reuses the trunk-path routing algorithm from the landscape canvas for clean connection lines between columns.

## Road Items

- ROAD-049 (Organization Intelligence Views)

## Acceptance Criteria

1. The User Type Journey page displays a multi-column flow with columns for User Types, User Stories, Capabilities, Bounded Contexts/Systems, and Teams
2. Clicking a user type node highlights its full chain through stories, capabilities, systems, and the teams that own them
3. Non-highlighted elements dim to 0.25 opacity when a user type is selected
4. Broken chains are visually indicated: capabilities without implementing systems show a warning icon, stories without capability links show a gap indicator
5. The flow renders connection lines using the existing trunk-path routing algorithm for clean visual paths
6. A search/filter bar allows filtering by user type name or team name
7. The journey view loads within 3 seconds for up to 10 user types with 50 stories
