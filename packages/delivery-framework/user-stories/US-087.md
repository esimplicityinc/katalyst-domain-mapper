# US-087: Receive Real-Time Scan Progress Events

**As an** Engineering Team Lead,
**I want to** receive real-time progress events during a scan,
**so that** I can see which specialist agent is currently running and how long the scan is taking.

## Context

The LangGraph scan adapter uses a StateGraph with fan-out to 5 specialist ReAct agents (CI, tests, architecture, domain, documentation) running in parallel. The `stream("updates")` mode emits events for each node transition, which can be forwarded to the client via SSE. This complements the existing SSE streaming capability (CAP-008) with scan-specific progress tracking that shows per-agent status rather than just raw text output.

## Road Items

- (partial — LangGraph adapter implements streaming, UI wiring is incomplete)

## Acceptance Criteria

1. The LangGraph scan adapter emits `ScanProgressEvent` objects for each node transition (detectContext, specialist fan-out, synthesize)
2. Progress events include `agentName`, `status` (`running`, `completed`, `failed`), and `timestamp`
3. The SSE streaming endpoint delivers progress events to the web UI in real-time
4. The UI displays a progress indicator showing which of the 5 specialist agents has completed
5. If a specialist agent fails, the progress event includes the error context and the scan continues with remaining agents
