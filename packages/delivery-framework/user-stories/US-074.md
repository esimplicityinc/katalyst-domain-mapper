# US-074: Filter Landscape by User Type

**As an** Engineering Team Lead,
**I want to** filter the landscape by user type,
**so that** I can trace how a specific user type flows through the system from their stories to the capabilities they depend on.

## Context

The landscape canvas can become visually overwhelming with many user types and workflows active simultaneously. The user type filter allows focusing on a single user's journey — dimming everything unrelated and highlighting their story-to-capability connections in the user type's color. Three bugs were fixed during initial implementation: non-selected groups were disappearing (should dim, not hide), story counts in collapsed labels were incorrect, and non-selected workflow dots were not being hidden.

## Road Items

- [ROAD-044](../roads/ROAD-044) — Business Landscape Graph

## Acceptance Criteria

1. Clicking a user type badge activates a user type filter — the selected user type's story-to-capability connection lines render in the user type's color
2. Non-selected elements dim to 0.25 opacity when a user type filter is active
3. The active filter is displayed as a pill in the toolbar with a clear button
4. Clicking the same user type badge again clears the filter and restores full opacity
5. When a user type filter is active, only workflows whose capabilities overlap with the user type's stories are shown
6. Collapsed user type groups show accurate story counts (not capability counts)
