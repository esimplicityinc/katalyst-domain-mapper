# US-085: Query Team Membership Across Multiple Teams

**As a** Domain Architect,
**I want to** query persons who serve on multiple teams,
**so that** I can identify cross-cutting concerns, cognitive load risks, and Team Topologies alignment issues.

## Context

Team Topologies recommends limiting cognitive load by keeping teams focused. When persons serve on multiple teams, it signals potential organizational coupling or insufficient team staffing. The persons API endpoint returns team associations, enabling architects to identify multi-team memberships and assess whether the organizational structure aligns with domain boundaries visible in the Business Landscape.

## Acceptance Criteria

1. `GET /api/v1/taxonomy/persons` returns all persons with their team associations
2. Multi-team membership is visible (a person appearing in multiple teams' member lists)
3. The landscape visualization shows team ownership on bounded context nodes
4. Team type badges (`stream-aligned`, `platform`, `enabling`, `complicated-subsystem`) render in the landscape
