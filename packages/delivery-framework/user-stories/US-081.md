# US-081: Toggle Features Via Flags Without Redeployment

**As a** Platform Engineer,
**I want to** control feature availability through configuration flags without redeploying the application,
**so that** I can safely roll out new features incrementally and quickly disable problematic ones.

## Context

Feature flags are core infrastructure — 9 flags currently control scanner availability, AI runtime selection, landscape layout engines, governance features, and concurrent scan limits. The `@foe/feature-flags` package provides a vendor-agnostic implementation built on the OpenFeature standard, ensuring portability if the team later adopts a commercial flag provider (LaunchDarkly, Flagsmith, etc.).

## Road Items

- (retroactive — built as cross-cutting infrastructure)

## Acceptance Criteria

1. A `flags.json` file defines all available flags with `key`, `defaultValue`, and `description`
2. Server-side flag evaluation is available via `initServerFlags()` using the OpenFeature SDK
3. Client-side flags are fetched from `GET /api/v1/flags` and initialized in the React app
4. Flag values support `boolean`, `string`, and `number` types
5. The `useFlag` hook provides reactive flag values in React components
6. Adding a new flag requires only a `flags.json` entry — no code changes
