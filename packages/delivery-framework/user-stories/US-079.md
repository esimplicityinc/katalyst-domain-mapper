# US-079: Start Full Dev Environment in One Command

**As a** Platform Engineer,
**I want to** start the full development environment with a single command and have BDD tests automatically check server health before running,
**so that** I never encounter cryptic test timeouts because a server wasn't started.

## Context

Before this work, starting the Intelligence package's dev environment required knowing to run two separate commands in two terminals, and BDD test failures in an unstarted environment produced confusing timeout errors. The `just dev` recipe, `just dev-ready` health check, and the mandatory dev-ready gating on all BDD recipes eliminate this entire class of developer friction.

## Road Items

- [ROAD-047](../roads/ROAD-047) — Developer Experience: One-Command Dev Environment

## Acceptance Criteria

1. `just dev` starts both the Intelligence API (port 3001) and Vite frontend (port 3002) in parallel
2. `just dev-ready` exits 0 within 3 seconds if both ports are accepting connections
3. `just dev-ready` exits 1 with actionable instructions naming the exact command to run if either server is down
4. All BDD recipes automatically invoke `dev-ready` before executing and fail immediately with a clear message if the environment is not ready
5. `just dev-status` displays Docker Compose service status and port reachability for both API and frontend
6. The pre-push hook gracefully skips BDD smoke tests when the dev server is not running
