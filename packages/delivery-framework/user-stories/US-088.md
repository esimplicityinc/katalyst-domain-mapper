# US-088: Chat Sessions Persist Conversation History

**As an** Engineering Team Lead,
**I want** my chat conversations with the FOE assessment coach to persist across page navigations,
**so that** I can continue a discussion about my scan results without losing context.

## Context

The LangGraph chat adapter uses `MemorySaver` for turn-by-turn conversation persistence. Each conversation is identified by a `threadId` that the client passes on subsequent messages, enabling the LLM to maintain full context. The frontend chat component needs to persist the `threadId` and replay the conversation history when the tab is revisited. This applies across all three chat integrations (FOE, DDD, Taxonomy) via the shared `@katalyst/chat` package.

## Road Items

- (partial — backend MemorySaver exists, frontend session restoration is incomplete)

## Acceptance Criteria

1. The LangGraph chat adapter uses `MemorySaver` to persist conversation state across turns
2. A `threadId` identifies each conversation and is passed on subsequent messages
3. Navigating away from and back to the chat tab restores the conversation history
4. The chat component displays the full conversation history on mount
5. A "New Conversation" action clears the history and starts a fresh thread
