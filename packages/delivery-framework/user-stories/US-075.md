# US-075: Lint Domain Model Landscape for Coverage Gaps

**As an** Engineering Team Lead,
**I want to** run a linter against my domain landscape,
**so that** I can identify coverage gaps, broken cross-references, and modeling inconsistencies before they compound into larger architectural problems.

## Context

Domain models are rarely complete — capabilities get added without corresponding bounded contexts, events proliferate without declared consumers, user types accumulate without user stories. The Landscape Domain Linter (CAP-024) provides a systematic quality gate: a pure-function domain layer computing 7 quantitative coverage metrics and running 3 rule sets (referential, semantic, coverage) against the full landscape graph.

## Road Items

- [ROAD-044](../roads/ROAD-044) — Business Landscape Graph

## Acceptance Criteria

1. The landscape linter computes all 7 coverage metrics: userTypeToStory, storyToCapability, capabilityToContext, contextToEvent, eventToConsumer, workflowToContext, eventToCapability
2. Referential integrity rules detect broken cross-references between governance artifacts
3. Semantic rules detect naming/modeling convention violations
4. Coverage rules surface completeness gaps (e.g., capabilities with no bounded context, events with no consumer)
5. Lint findings include severity (error/warning/info) and category classification
6. The LandscapeLinter is a pure function with zero I/O — fully testable in isolation
