# US-093: Manage Practice Areas via CRUD API

**As a** Platform Engineer,
**I want to** manage practice areas through the CRUD API,
**so that** I can define and maintain the maturity domains that teams are assessed against.

## Context

Practice areas are the top-level maturity domains in the adoption tracking system. Each practice area contains six pillars (strategy, standards, frameworks, libraries, processes, measures) that define what it means to adopt that practice. The CRUD API enables programmatic management of practice areas as part of the broader taxonomy system.

## Road Items

- ROAD-048 (Practice Area, Competency & Adoption CRUD)

## Acceptance Criteria

1. GET `/api/v1/taxonomy/practice-areas` returns a list of all practice areas with their pillar structures
2. GET `/api/v1/taxonomy/practice-areas/:name` returns a single practice area by name with full detail
3. POST `/api/v1/taxonomy/practice-areas` creates a new practice area with validated six-pillar structure (strategy, standards, frameworks, libraries, processes, measures)
4. PUT `/api/v1/taxonomy/practice-areas/:name` updates an existing practice area and persists changes
5. DELETE `/api/v1/taxonomy/practice-areas/:name` removes a practice area from the system
