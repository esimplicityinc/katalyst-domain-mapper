# US-083: Universal Contribution Lifecycle for Governance Artifacts

**As a** Documentation Author,
**I want** all governance artifact types to follow a universal contribution lifecycle (draft → proposed → accepted → deprecated → superseded),
**so that** I don't need to learn different state machines for different artifact types and validation is consistent.

## Context

The taxonomy consolidation unified the formerly separate `ddd/` and `governance/` schema modules into a single `taxonomy/` module. As part of this work, a universal `ContributionSchema` was added to all 13 extension types, replacing ad-hoc status fields with a consistent 5-state lifecycle and validated transition rules. This ensures that ADRs, capabilities, user stories, aggregates, bounded contexts, and all other taxonomy artifacts follow the same governance workflow.

## Road Items

- (retroactive — built during taxonomy consolidation)

## Acceptance Criteria

1. The `ContributionSchema` defines 5 states: `draft`, `proposed`, `accepted`, `deprecated`, `superseded`
2. The `ContributionTransitionsSchema` enforces valid transitions (e.g., `draft` can move to `proposed` or `deprecated`, but not directly to `accepted`)
3. All 13 taxonomy extension schemas (ADR, aggregate, capability, bounded-context, etc.) include a `contribution` field conforming to this lifecycle
4. Invalid state transitions are rejected at the schema validation boundary
5. The `supersededBy` field creates a traceable chain from deprecated artifacts to their replacements
