# US-028: Create and Manage DDD Domain Models via API

**As a** Platform Engineer,
**I want to** create and manage DDD domain models via the REST API,
**so that** domain knowledge is captured in a structured, queryable format that integrates with governance validation and visualization.

## Context

The domain-models API provides CRUD operations for the full DDD artifact hierarchy: domain model workspaces contain bounded contexts, aggregates, value objects, domain events, and glossary terms. This structured representation enables the DDD context map visualization (ROAD-009) and integrates with governance scoring for the Understanding dimension.

## Road Items

- **ROAD-008** (Delivery Framework Integration) -- Domain models are part of the self-referential domain content
- **ROAD-009** (Governance & DDD Web Visualization) -- Context map consumes domain model data

## Acceptance Criteria

1. `POST /api/v1/domain-models` creates a workspace with a generated UUID
2. `GET /api/v1/domain-models` returns an array of all models
3. `GET /api/v1/domain-models/:id` returns the model with nested `boundedContexts`, `aggregates`, `valueObjects`, `domainEvents`, `glossaryTerms`
4. `DELETE /api/v1/domain-models/:id` cascades deletion to all child artifacts
5. Child artifact endpoints (contexts, aggregates, events, value-objects, glossary) return the created item with a generated UUID
6. `GET /api/v1/domain-models/nonexistent-id` returns 404
