# US-080: Use Shared Chat Component Across AI Pages

**As a** Platform Engineer,
**I want to** use a shared, configurable chat component library for all AI-powered pages,
**so that** I don't duplicate chat UI code across FOE, DDD, and Taxonomy pages, and every chat experience is visually consistent.

## Context

Before the `@katalyst/chat` extraction, each AI-powered page (FOE assessment, DDD modeling, Taxonomy management) contained its own copy of chat state management, SSE streaming logic, and markdown rendering. The shared package eliminates ~500 lines of duplicated code and ensures all three chat experiences evolve together. The component supports agent targeting via `agentId`, theme matching via `accentColor`, and suggestion chips for guided exploration.

## Acceptance Criteria

1. The `@katalyst/chat` package exports `OpenCodeChat` (component) and `useOpenCodeChat` (hook)
2. `OpenCodeChat` accepts an `agentId` prop to target different AI agents (e.g., `foe-assessment-coach`, `ddd-domain-mapper`, `taxonomy-architect`)
3. `OpenCodeChat` supports `accentColor` variants (`purple`, `blue`, `green`, `rose`) to match each page's theme
4. Chat messages render markdown with syntax highlighting
5. The hook manages SSE streaming, session creation, and question/answer tool handling without page-specific code
6. FOEChat, DDDChat, and TaxonomyChat all consume `@katalyst/chat` with zero duplicated logic
