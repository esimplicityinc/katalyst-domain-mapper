# US-077: Manage User Types and User Stories via UI

**As an** Engineering Team Lead,
**I want to** manage user types and user stories through a web UI,
**so that** I can maintain the human-centered view of our system alongside the technical capability taxonomy.

## Context

UserTypeListView (798 lines) and UserStoryBoardView (739 lines) provide full CRUD for the two governance artifact types that bridge business intent and technical capability. The kanban board pattern for user stories makes it easy to see the distribution across the story lifecycle and identify bottlenecks (e.g., too many stories stuck in "draft").

## Road Items

- [ROAD-046](../roads/ROAD-046) — Governance Artifact Management UI

## Acceptance Criteria

1. The User Types page displays all user types as cards with archetype badge, type icon, goals, pain points, behaviors, and story count
2. Users can create, edit, and delete user types with all fields including archetype, goals, pain points, behaviors, and typical capabilities
3. The Stories tab displays user stories as a kanban board grouped by status (draft → approved → implementing → complete → deprecated)
4. Users can create, edit, and delete user stories with user type linking and capability multi-select
5. Story creation requires selecting at least one capability
6. All changes persist immediately to the API
