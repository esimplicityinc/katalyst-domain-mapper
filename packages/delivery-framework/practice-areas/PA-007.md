# Documentation

The Documentation practice area covers all aspects of knowledge capture and distribution for engineering teams. It encompasses architecture decision records (ADRs), API documentation, runbooks, onboarding guides, and living documentation practices.

Teams practicing at the **mastered** level treat documentation as a first-class engineering artifact, maintain it alongside code, and use it as a primary mechanism for knowledge distribution and organizational resilience.
