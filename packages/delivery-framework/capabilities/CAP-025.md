# CAP-025: Taxonomy & User Type Management UI

## Description

The Taxonomy & User Type Management UI provides complete CRUD interfaces for the three core governance artifact types that feed the Business Landscape: capabilities (in a 3-level hierarchy), user types (with archetype, goals, pain points, behaviors), and user stories (in a kanban-style board by status). Two conversational AI agents — `taxonomy-architect` and `user-type-storyteller` — are embedded as chat tabs within each management page, enabling guided discovery and automatic real-time persistence of artifacts to the API. New capabilities appear in the landscape immediately without requiring a page reload.

## Key Features

- **CapabilityTreeView** (813 lines): Interactive 3-level capability hierarchy tree. Add/edit/delete capabilities. Nodes show status badges (planned/stable/deprecated), taxonomy node associations, and parentCapability links. Full inline edit form.
- **UserTypeListView** (798 lines): Card-based list of user types with full CRUD. Each card shows archetype badge, type icon, goals, pain points, behaviors, and story count. Inline create/edit form.
- **UserStoryBoardView** (739 lines): Kanban board grouped by status (draft → approved → implementing → complete → deprecated). Full CRUD with user type linking and capability multi-select.
- **TaxonomyChat** (698 lines): SSE-streaming AI chat wired to `taxonomy-architect` agent. Pre-loads current taxonomy snapshot and capability tree as system context. Auto-persists via curl.
- **UserTypeChat** (697 lines): SSE-streaming AI chat wired to `user-type-storyteller` agent. Implements formal 6-phase discovery protocol (see ADR-019).
- **Database migration 0009**: `governance_user_types` and `governance_user_stories` tables; expanded `governance_capabilities` columns (parent, description, category, dependsOn)
- **9 new API routes**: GET/POST/PUT/DELETE for /api/v1/governance/capabilities, /user types, /user-stories

## Architecture

```
ArchitecturePage (/design/architecture/*)
    ├── Systems tab (taxonomy read-only)
    ├── Capabilities tab → CapabilityTreeView
    └── Chat tab → TaxonomyChat → taxonomy-architect agent

UserTypesPage (/design/user-types/*)
    ├── User Types tab → UserTypeListView
    ├── Stories tab → UserStoryBoardView
    └── Chat tab → UserTypeChat → user-type-storyteller agent

API: /api/v1/governance/capabilities | user types | user-stories
DB: governance_capabilities | governance_user_types | governance_user_stories
```

## NFR Requirements

- CRUD operations complete `< 200ms` (SQLite operations)
- AI chat first token `< 2s` (SSE streaming)
- Capability tree renders up to 50 nodes without performance degradation
- Kanban board renders up to 200 stories without virtualization

## User Types

- **UT-001** (Engineering Team Lead) — Manages capabilities to keep the taxonomy reflecting real system evolution, and reviews AI-generated capability suggestions from the taxonomy-architect agent before accepting them
- **UT-002** (Platform Engineer) — Facilitates discovery workshops using the user-type-storyteller agent's 6-phase protocol to elicit and persist user types and stories directly from domain expert conversations
- **UT-007** (Domain Modeler) — Maps capabilities to taxonomy nodes via the CapabilityTreeView to ensure the landscape visualization accurately reflects system boundaries

## Related Capabilities

- **CAP-019** (Taxonomy CRUD API) — Underlying taxonomy snapshot infrastructure that the capability tree builds upon and the TaxonomyChat agent reads as system context
