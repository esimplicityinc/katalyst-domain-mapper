# CAP-033: Organization & Team Intelligence

## Description

Organization & Team Intelligence provides interactive web UI views that surface the relationships between people, teams, systems, and user types. The centerpiece is a force-directed Organization Map graph that visualizes the structural spine of the engineering organization — who is on which teams, which teams own which systems, and which user types those systems serve. Complementing the graph are detail pages for teams (members, owned systems, users served, practice area adoption levels) and persons (team memberships, competency radar chart, skill assessments, contribution activity timeline).

## Key Features

- **Organization Map Graph**: Force-directed SVG canvas (d3-force) showing teams, persons, systems/bounded contexts, and user types as interconnected nodes with membership, ownership, and serves edges
- **Team Profile Page**: Detail view for a single team with 4 panels — members (with role badges), owned systems, served user types (derived through capability linkage), and practice area adoption progress bars
- **Person Competency Profile**: Detail view for an individual with team memberships, a recharts RadarChart plotting skill gauges across practice areas, and a contribution activity timeline
- **Team List View**: Card-based overview of all teams with Team Topologies type badges, member counts, and owned system counts
- **People List View**: Card-based overview of all persons with team badges and adoption summary

## Architecture

```
Pages:
├── /organization              (landing with tool cards)
├── /organization/overview     (OrgMapCanvas — d3-force graph)
├── /organization/teams        (TeamsListView — card list)
├── /organization/teams/:name  (TeamProfilePage — detail)
├── /organization/people       (PeopleListView — card list)
└── /organization/people/:name (PersonProfilePage — detail with radar)

Components:
├── OrgMapCanvas.tsx           (d3-force SVG, reuses useSvgPanZoom)
├── TeamsListView.tsx          (team cards with type badges)
├── TeamProfilePage.tsx        (4-panel detail)
├── PeopleListView.tsx         (person cards with team badges)
└── PersonProfilePage.tsx      (radar chart + contribution timeline)

API (new aggregate endpoints):
├── GET /api/v1/taxonomy/teams/:name/profile
└── GET /api/v1/taxonomy/persons/:name/profile
```

## User Types

- **UT-007** (Domain Architect) — Uses the org map to identify Conway's Law alignment issues, cognitive load risks from multi-team membership, and team-to-system ownership gaps
- **UT-001** (Engineering Team Lead) — Uses team profiles to assess member composition and practice area adoption, and person profiles to plan individual development
- **UT-006** (Non-Technical Stakeholder) — Uses the org map and team profiles to understand organizational structure without reading code
