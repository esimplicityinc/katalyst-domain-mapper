# CAP-035: Value Stream Traceability

## Description

Value Stream Traceability provides two complementary views that answer the questions "who are we building for and which teams deliver it?" and "which contributions led to what outcomes?" The User Type Journey view traces how user types flow through user stories to capabilities to bounded contexts to owning teams, displayed as a multi-column swim lane or Sankey-style flow. The Outcome Traceability view traces the reverse direction — from people through their contributions to road items to capabilities to user stories to the user types who receive value, displayed as a tree/cascade visualization.

## Key Features

- **User Type Journey Board**: Multi-column flow (User Type → User Stories → Capabilities → Bounded Contexts → Teams) with click-to-highlight for a single user type's chain
- **Outcome Traceability View**: Reverse cascade (Person → Contributions → Road Items → Capabilities → User Stories → User Types) showing how engineering investments deliver user value
- **Cross-Highlighting**: Click any node in either view to highlight its full upstream and downstream chain
- **Filtering**: Filter by user type, team, person, road item, or capability to focus on specific value streams
- **Gap Detection**: Visual indicators for broken chains (capabilities without implementing systems, stories without capability links, road items without contributions)

## Architecture

```

├── /strategy/value-streams              (landing with view selector)
├── /strategy/value-streams/journeys     (UserTypeJourneyView)
└── /strategy/value-streams/traceability (OutcomeTraceView)

Components:
├── UserTypeJourneyView.tsx    (multi-column SVG with trunk-path routing)
├── JourneyColumn.tsx          (single column: user types, stories, capabilities, systems, or teams)
├── OutcomeTraceView.tsx       (tree/cascade SVG)
└── TraceChain.tsx             (single traced path with status indicators)

Data Sources (may need new join endpoint):
├── GET /api/v1/taxonomy/snapshots/:id   (full snapshot with nodes, extensions)
├── GET /api/v1/taxonomy/teams           (teams with owned nodes)
├── GET /api/v1/governance/snapshots/:id (capabilities, user types, stories, road items)
└── GET /api/v1/contributions            (contribution items with author info)
```
