# CAP-032: Practice Area & Competency Management

## Description

Practice Area & Competency Management introduces practice areas, competencies, team adoptions, and individual adoptions as first-class entities in the taxonomy system. Practice areas represent top-level maturity domains (e.g., "Continuous Delivery", "Domain-Driven Design") with six standard pillars: strategy, standards, frameworks, libraries, processes, and measures. Competencies are assessable skills within a practice area, supporting three proficiency levels (basic, intermediate, advanced) and directed acyclic graph (DAG) dependencies between prerequisites. Team adoptions track team-level maturity per practice area, while individual adoptions track person-level skill assessments with evidence and notes.

## Key Features

- **Practice Area CRUD**: Full create, read, update, delete for practice areas with six-pillar structure (strategy, standards, frameworks, libraries, processes, measures)
- **Competency CRUD**: Manage assessable skills within practice areas with three proficiency levels (basic, intermediate, advanced) and DAG prerequisite dependencies
- **Team Adoption Tracking**: Record and query team-level maturity for each practice area with maturity levels, evidence, and assessment dates
- **Individual Adoption Tracking**: Record and query person-level skill assessments per practice area with competency level, evidence, and notes
- **22 API Endpoints**: Comprehensive REST API across four route groups
- **DAG Validation**: Competency prerequisite chains are validated to prevent circular dependencies

## Architecture

```
Schemas:
├── TaxonomyPracticeAreaSchema     (name, displayName, description, pillars, status)
├── TaxonomyCompetencySchema       (name, displayName, practiceAreaName, level, prerequisites)
├── TaxonomyTeamAdoptionSchema     (teamName, practiceAreaName, maturityLevel, evidence, assessedAt)
└── TaxonomyIndividualAdoptionSchema (personName, practiceAreaName, competencyLevel, evidence, notes)

Database:
├── taxonomy_practice_areas
├── taxonomy_competencies
├── taxonomy_team_adoptions
└── taxonomy_individual_adoptions

API Routes (22 endpoints):
├── Practice Areas:
│   ├── GET    /api/v1/taxonomy/practice-areas
│   ├── GET    /api/v1/taxonomy/practice-areas/:name
│   ├── POST   /api/v1/taxonomy/practice-areas
│   ├── PUT    /api/v1/taxonomy/practice-areas/:name
│   └── DELETE /api/v1/taxonomy/practice-areas/:name
├── Competencies:
│   ├── GET    /api/v1/taxonomy/competencies
│   ├── GET    /api/v1/taxonomy/competencies/:name
│   ├── POST   /api/v1/taxonomy/competencies
│   ├── PUT    /api/v1/taxonomy/competencies/:name
│   └── DELETE /api/v1/taxonomy/competencies/:name
├── Team Adoptions:
│   ├── GET    /api/v1/taxonomy/team-adoptions
│   ├── GET    /api/v1/taxonomy/team-adoptions/:id
│   ├── POST   /api/v1/taxonomy/team-adoptions
│   ├── PUT    /api/v1/taxonomy/team-adoptions/:id
│   ├── DELETE /api/v1/taxonomy/team-adoptions/:id
│   └── GET    /api/v1/taxonomy/team-adoptions/by-team/:teamName
├── Individual Adoptions:
│   ├── GET    /api/v1/taxonomy/individual-adoptions
│   ├── GET    /api/v1/taxonomy/individual-adoptions/:id

│   ├── DELETE /api/v1/taxonomy/individual-adoptions/:id
│   └── GET    /api/v1/taxonomy/individual-adoptions/by-person/:personName
```
