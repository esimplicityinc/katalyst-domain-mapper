# CAP-030: Switchable AI Runtime Orchestration

## Description

The Switchable AI Runtime Orchestration capability provides a hexagonal port/adapter architecture that abstracts AI runtime selection behind clean interfaces. The `ScanOrchestrator` and `ChatOrchestrator` ports define runtime-agnostic contracts, while concrete adapters implement them for OpenCode (Docker CLI-based) and LangGraph (StateGraph-based) runtimes. An `orchestratorFactory` reads `scan.runtime` and `chat.runtime` feature flags at startup and dynamically imports the correct adapter. If LangGraph is selected but no LLM API key is available, the system gracefully falls back to OpenCode.

## Key Features

- **`ScanOrchestrator` port**: Defines `startScan(context)`, `getProgress()`, and `getResult()` — returns `ScanOrchestrationResult` regardless of runtime
- **`ChatOrchestrator` port**: Defines `sendMessage(options)` with `ChatStreamEvent` streaming — runtime-transparent conversation interface
- **`orchestratorFactory`**: Reads feature flags, dynamically imports the correct adapter, caches instances for reuse
- **LangGraph adapter**: Fan-out to 5 specialist ReAct agents (CI, tests, arch, domain, docs) in parallel, fan-in to synthesizer; `MemorySaver` for chat persistence; streaming progress events
- **OpenCode adapter**: Docker container execution with stdout capture
- **Status endpoint**: `GET /api/v1/orchestrator/status` returns active runtimes, supported runtimes, consistency check, and health status
- **Graceful fallback**: Missing API key triggers automatic fallback from LangGraph to OpenCode with logged warning

## Architecture

```
Ports (interfaces):
├── ScanOrchestrator.ts   (AgentRuntime, ScanContext, ScanProgressEvent, ScanOrchestrationResult)
└── ChatOrchestrator.ts   (ChatStreamEvent, SendMessageOptions)

Adapters:
├── opencode/
│   ├── OpenCodeScanOrchestrator.ts
│   └── OpenCodeChatOrchestrator.ts
└── langgraph/
    ├── LangGraphScanOrchestrator.ts
    ├── LangGraphChatOrchestrator.ts
    ├── graphs/scanGraph.ts    (StateGraph: detectContext → specialists → synthesize)
    ├── graphs/chatGraph.ts    (ReAct agent + MemorySaver)
    ├── nodes/specialists.ts   (5 parallel ReAct agents with tools)
    ├── nodes/prompts.ts       (shared prompt loading from .opencode/agents/)
    └── tools/repoTools.ts     (sandboxed file system tools)

Factory:
└── orchestratorFactory.ts    (reads flags → imports adapter → caches)

API:
└── GET /api/v1/orchestrator/status
```
