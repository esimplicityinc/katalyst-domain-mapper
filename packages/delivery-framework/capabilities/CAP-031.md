# CAP-031: Layer Health Measurement

## Description

Layer Health Measurement provides an automated quality scoring system for each layer in the system taxonomy. Health is measured across three dimensions: **Understandability** (lint errors, formatting violations, documentation coverage), **Functionality** (test pass/fail/skip counts, code coverage), and **Compliance** (vulnerability counts by severity, license issues, dependency audit results). Each dimension carries a 0-100 score, a pass/warn/fail status, typed metrics, and references to the governing NFRs and ADRs. A weighted composite `overallScore` and worst-category `overallStatus` provide a quick health summary.

## Key Features

- **Three measurement categories**: Understandability, Functionality, Compliance — each with typed metrics
- **Score and status per category**: `score` (0-100), `status` (pass/warn/fail), `metrics` array with measurement-specific types
- **NFR/ADR traceability**: Each category links to `nfrIds` and `adrIds` explaining what standards define "pass"
- **Overall composite**: `overallScore` (weighted average), `overallStatus` (worst-performing category wins)
- **Taxonomy integration**: `layerHealths` collection in `TaxonomySnapshotSchema` — health travels with taxonomy data

## Schema Structure

```
LayerHealth
├── layerName: string
├── overallScore: number (0-100)
├── overallStatus: "pass" | "warn" | "fail"
├── understandability:
│   ├── score, status, nfrIds, adrIds
│   └── metrics: [{ type: "lint_errors" | "format_violations" | "doc_coverage", ... }]
├── functionality:
│   ├── score, status, nfrIds, adrIds
│   └── metrics: [{ type: "unit_test" | "integration_test" | "coverage", ... }]
└── compliance:
    ├── score, status, nfrIds, adrIds
    └── metrics: [{ type: "vulnerabilities" | "license_issues" | "dependency_audit", ... }]
```
