# CAP-028: Feature Flag Management

## Description

The `@foe/feature-flags` package provides a vendor-agnostic feature flag management system built on the OpenFeature standard. It supports server-side evaluation via `initServerFlags()`, client-side consumption via React hooks (`useFlag`, `useBooleanFlagValue`), and CI/CD-friendly environment variable overrides (`FF_*` prefix). Nine flags currently control scanner availability, AI runtime selection, landscape layout engines, governance features, and more.

## Key Features

- **`flags.json` configuration**: Single source of truth for all flag definitions with `key`, `defaultValue`, and `description`
- **Server-side evaluation**: `initServerFlags()` reads `flags.json`, applies `FF_*` env var overrides, registers an OpenFeature InMemoryProvider
- **Client-side consumption**: `fetchAndInitFlags()` fetches from `GET /api/v1/flags`, initializes OpenFeature React SDK; re-exports `useFlag`, `useBooleanFlagValue`, `useStringFlagValue`, `useNumberFlagValue`
- **Environment variable overrides**: `FF_SCANNER_ENABLED=false` overrides `scanner-enabled` flag; kebab-to-SCREAMING_SNAKE conversion
- **Typed flag keys**: `FlagKeys` const provides autocomplete for all 9 flags
- **API endpoint**: `GET /api/v1/flags` returns all flag values for client initialization

## Current Flags

| Key | Type | Default | Purpose |
|-----|------|---------|---------|
| `scanner-enabled` | boolean | true | Enable/disable scanner |
| `domain-model-v2` | boolean | true | V2 domain model features |
| `landscape-lint-strict` | boolean | false | Strict landscape linting |
| `ai-insights-enabled` | boolean | true | AI insight generation |
| `governance-v2` | boolean | false | V2 governance features |
| `max-concurrent-scans` | number | 3 | Concurrent scan limit |
| `scan.runtime` | string | "langgraph" | Scan orchestrator runtime |
| `chat.runtime` | string | "langgraph" | Chat orchestrator runtime |
| `landscape-layout-v2` | boolean | false | V2 zoned layout engine |

## Architecture

```
@foe/feature-flags
├── flags.json              (flag definitions)
├── src/flags.ts            (FlagKeys, types)
├── src/server.ts           (initServerFlags, evaluateAllFlags)
├── src/client.ts           (fetchAndInitFlags, React hooks)
└── __tests__/              (unit tests)

API integration:
└── GET /api/v1/flags → evaluateAllFlags() → JSON response
```
