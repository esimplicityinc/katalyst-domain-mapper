# CAP-023: Business Landscape Visualization

## Description

The Business Landscape canvas provides a live, interactive SVG visualization of the entire domain's interconnections. It fetches data from the `GET /api/v1/landscape/:domainModelId` endpoint and renders: taxonomy-grouped bounded context boxes with depth-coded colors, directed domain event flow arrows with workflow-filter awareness, animated user type dots riding workflow event chains via `animateMotion`, capability nodes as port-style connectors, user type badges with click-to-filter, user story connection lines that highlight in user type color when filtered, and auto-detected "unknown external system" dashed boxes inferred from unresolved event consumers.

## Key Features

- **10-layer composited SVG canvas**: Grid background → system hierarchy boxes → event flow arrows → workflow chain paths → animated user type dots → story connection lines → story dots (on filter) → capability nodes → user type badges → inferred system boxes
- **Three interchangeable layout engines** (ADR-018): ELK (Eclipse Layout Kernel, default — taxonomy-hierarchical), Dagre (DAG layout), D3-Force (physics simulation) — switchable at runtime via toolbar
- **User Type filter**: Click a user type badge to filter the landscape — non-selected elements dim to 0.25 opacity; selected user type's story→capability lines render in user type color at 0.8 opacity
- **Workflow filter**: Dropdown to show only specific workflow event chains; active workflow edges rendered with glow filter
- **Collapse/expand user types**: User Type groups with multiple stories can be collapsed into a single count badge; `useCollapseAnimation` hook provides 300ms cubic-eased transitions
- **Layout time display**: Toolbar shows engine execution time in ms, enabling informed engine selection for large graphs
- **Inferred external systems**: Unknown `consumedBy` references auto-detected and rendered as dashed "???" boxes

## Architecture

```
LandscapeView.tsx (data-fetch + state)
    ├── GET /api/v1/landscape/:modelId
    ├── LandscapeLayoutEngine (interface)
    │   ├── ElkLayoutEngine.ts (ELK, taxonomy-aware, default)
    │   ├── DagreLayoutEngine.ts (DAG layout)
    │   └── D3ForceLayoutEngine.ts (physics)
    └── LandscapeCanvas.tsx (10-layer SVG renderer)
        └── useCollapseAnimation.ts (RAF-based transitions)
```

## NFR Requirements

- Layout computation < 500ms for graphs up to 50 bounded contexts
- SVG rendering < 100ms after layout completes
- User Type filter toggle response < 50ms (client-side state change only)
- Collapse/expand animation runs at 60fps (requestAnimationFrame-based)

## User Types

- **UT-001** (Engineering Team Lead) — Uses the landscape canvas to get a holistic view of domain interconnections, identify coupling hotspots, and communicate system structure to stakeholders
- **UT-007** (Domain Modeler) — Uses the visualization to validate event flows, verify bounded context boundaries, and discover gaps in the domain model surfaced by inferred external system boxes
- **UT-006** (Non-Technical Stakeholder) — Uses the user type filter and animated workflow flows to understand how specific user journeys traverse the system without needing to read code or architecture diagrams
