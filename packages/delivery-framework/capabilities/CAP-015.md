# CAP-015: Manual FOE Assessment Creation

## Description

The system provides a step-by-step wizard interface that enables users to create FOE assessment reports manually. This capability supports scenarios where automated scanning is not feasible or desirable, such as assessments without code access, initial baseline assessments, or educational use cases.

## Key Features

- **7-Step Wizard Interface**: Metadata → 3 Dimensions → Findings → Recommendations → Review
- **Visual Dimension Scoring**: Interactive sliders with real-time feedback (0-100 scale)
- **Assessment Guidance**: Built-in criteria and scoring rubrics for each dimension
- **Automatic Calculations**: Overall score computed from weighted dimensions
- **JSON Export**: Generates valid FOE report JSON matching scanner output
- **Template Integration**: Exported reports work in all 11 visualization templates

## Architecture

```
Web UI (React Components)
    ↓
AssessmentWizard Component
    ├── MetadataStep
    ├── FeedbackDimensionStep (35% weight)
    ├── UnderstandingDimensionStep (35% weight)
    ├── ConfidenceDimensionStep (30% weight)
    ├── FindingsStep (strengths + gaps)
    ├── RecommendationsStep
    └── ReviewStep
    ↓
JSON Export (FOEReport schema)
    ↓
Visualization Templates (CAP-001, CAP-002)
```

## Personas

- **PER-001** (Engineering Team Lead) -- Creates manual assessments for their team
- **PER-005** (Independent Consultant) -- Uses wizard for client assessments without code access
- **PER-004** (Documentation Author) -- Educational tool for understanding FOE dimensions

## Use Cases

- **No Code Access**: Assess practices when repository access is restricted
- **Initial Baseline**: Quick assessment before investing in automated scanning
- **Educational**: Teaching tool for understanding FOE principles
- **Small Projects**: Manual assessment sufficient for small codebases (&lt;10k LOC)
- **Comparative Analysis**: Consistent scoring framework across teams/projects

## Non-Functional Requirements

### Usability (NFR-USAB-001)
- **Target**: Wizard completion time &lt;10 minutes for experienced users
- **Measurement**: User testing + analytics tracking
- **Rationale**: Manual assessment should be quick and efficient

### Accessibility (NFR-A11Y-001)
- **Target**: WCAG 2.1 AA compliant
- **Measurement**: Automated axe-core testing + manual keyboard navigation
- **Rationale**: Inclusive design for all users

### Data Integrity (NFR-REL-001)
- **Target**: 100% schema validation success rate
- **Measurement**: Zod schema validation in unit tests
- **Rationale**: Manual reports must integrate seamlessly with existing system

### Performance (NFR-PERF-001)
- **Target**: Form interactions respond &lt;100ms
- **Measurement**: React DevTools profiling
- **Rationale**: Real-time feedback critical for good UX
