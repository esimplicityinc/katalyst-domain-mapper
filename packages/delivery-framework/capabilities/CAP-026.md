# CAP-026: Developer Environment Tooling

## Description

The Developer Environment Tooling capability provides a unified, one-command developer experience for the Intelligence package's full-stack dev environment. Key additions: `just dev` starts both API (port 3001) and Vite frontend (port 3002) in parallel; `just dev-ready` provides a fast health check that gates BDD test execution on server availability; `just dev-status` gives a diagnostic snapshot of service health. All BDD recipes now automatically check environment readiness before executing, eliminating cryptic timeout failures. ESLint configuration was also hardened with missing browser globals (Observer APIs, animation frame APIs) that the new landscape components require.

## Key Features

- **`just dev`**: Starts Bun API (hot reload, port 3001) and Vite frontend (port 3002) in parallel in one command
- **`just dev-intelligence-api`**: Starts only the API server with hot reload
- **`just dev-intelligence-web`**: Starts only the Vite frontend (proxies `/api` → port 3001)
- **`just dev-ready`**: Fast health check — curls both ports, exits 0 if both up, exits 1 with actionable instructions if either is down
- **`just dev-status`**: Shows Docker Compose service status + port reachability for both services
- **BDD gating**: All BDD recipes (`bdd-api`, `bdd-ui`, `bdd-hybrid`, `bdd-tag`, `bdd-roadmap`, `bdd-headed`) invoke `dev-ready` as prerequisite — fail fast with clear message instead of timing out
- **ESLint browser globals**: Added `MutationObserver`, `ResizeObserver`, `IntersectionObserver`, `WheelEvent`, `Node`, `performance`, `requestAnimationFrame`, `cancelAnimationFrame` to eslint.config.js
- **Pre-push hook improvements**: Hooks gracefully skip BDD smoke tests when dev server is not running (CI environments without dev stack)

## Architecture

```
justfile
├── dev → dev-intelligence-api + dev-intelligence-web (parallel)
├── dev-intelligence-api → bun run --watch api/index.ts
├── dev-intelligence-web → vite (proxies /api → :3001)
├── dev-ready → curl :3001/health && curl :3002 → exit code
├── dev-status → docker compose ps + port checks
└── bdd-* → dev-ready (prerequisite) → katalyst run ...

.husky/pre-push
└── dev-ready check → skip BDD if server not running
```

## NFR Requirements

- `just dev-ready` completes in `< 3s`econds
- `just dev` brings both servers to "accepting connections" state in `< 15s`econds on warm node_modules
- BDD gate failure message provides exact command to start the dev environment

## User Types

- **UT-003** (Developer / Platform Engineer) — Uses `just dev` as daily driver to spin up the full stack with a single command, and relies on BDD gating to get an immediate actionable failure message rather than a hanging test suite
- **UT-002** (Platform Engineer) — Uses `just dev-status` to diagnose environment issues in CI and local environments, and benefits from pre-push hook improvements that prevent false failures in environments without a running dev stack
